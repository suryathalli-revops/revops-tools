Snowflake
Data Warehouse ¬∑ You already know this!
The single source of truth. All company data ‚Äî sales, product, finance ‚Äî lives and is queried here via SQL.
Use when
  • Running SQL queries on raw or transformed data
  • Exploring tables and schemas to understand data models
  • Building the data foundation Omni or dbt will use
  • Answering "what does the data actually say?" questions
üí° Your strongest existing skill ‚Äî lean on it early to build credibility.
üìä
Omni
BI & Visualization ¬∑ Replaces Looker
Business intelligence tool for building dashboards, charts, and self-serve analytics. Connects directly to Snowflake.
Use when
  • Building sales, marketing, or finance dashboards
  • Answering recurring business questions (pipeline, churn, CAC)
  • Sharing visual reports with non-technical stakeholders
  • Exploring data without writing SQL (drag-and-drop)
üí° Think of Omni as the "presentation layer" ‚Äî Snowflake holds the data, Omni makes it beautiful and accessible.
üî≠
Monte Carlo
Data Observability & Quality
Monitors your data pipelines for anomalies, missing data, schema changes, and freshness issues. Like an alarm system for your data.
Use when
  • A dashboard number "looks off" ‚Äî check here first
  • Investigating why a table hasn't updated
  • Setting up alerts on key metrics for anomalies
  • Understanding data lineage (where does this field come from?)
üí° Before escalating a data issue, check Monte Carlo ‚Äî it may already have the answer and root cause.
üíª
VSCode
Code Editor ¬∑ The Workspace
Your primary code editor. Write SQL, Python, dbt models, or any scripts here. The foundation everything else plugs into.
Use when
  • Writing or editing SQL queries, dbt models, Python scripts
  • Any file editing ‚Äî configs, CSVs, documentation
  • Using extensions like dbt Power User, SQL formatters
  • Basically: your all-day coding home base
üí° Start here. Everything else (Cursor, Git, Sourcegraph) integrates into or works alongside VSCode.
ü§ñ
Cursor
AI-Powered Code Editor ¬∑ VSCode + AI
VSCode but with a deeply integrated AI assistant. Use it to write, explain, debug, and refactor code with natural language.
Use when
  • Stuck on SQL syntax or a complex query ‚Äî ask it in plain English
  • Need to understand unfamiliar code (highlight + "explain this")
  • Writing boilerplate Python or data transformation logic
  • Debugging errors quickly
üí° As someone new to the stack, Cursor is your cheat code. Use it constantly to ramp up faster.
üî¨
Sourcegraph
Code Search & Navigation
Powerful search engine for Checkr's entire codebase. Find any SQL model, field definition, or transformation across all repos instantly.
Use when
  • "Where is the arr metric actually defined in the code?"
  • Finding all places a table or field is referenced
  • Understanding how a dbt model is built end-to-end
  • Onboarding ‚Äî exploring the codebase without cloning everything
üí° Use Sourcegraph before asking a teammate "where is X defined?" ‚Äî you'll find it in seconds.
ü¶ä
GitLab
Version Control Platform ¬∑ Code Home
Web platform where all code repositories live. Track changes, review pull requests, run CI/CD pipelines, and collaborate on code.
Use when
  • Reviewing or submitting changes to dbt models or scripts
  • Opening/reviewing merge requests (MRs) for code review
  • Checking pipeline run status after deploying a change
  • Browsing the history of who changed what and why
üí° GitLab is the website; Git (below) is the command-line tool you use locally to push changes there.
üåø
Git
Version Control ¬∑ Local Command Line
The command-line tool for tracking changes to your code locally. Think of it as "save + track history" for any file you edit.
Use when
  • Saving changes to a SQL file or dbt model (git commit)
  • Pushing your work up to GitLab (git push)
  • Creating a new branch before making changes (git checkout -b)
  • Pulling the latest code from teammates (git pull)
üí° Core Git commands you'll use daily: pull ‚Üí branch ‚Üí edit ‚Üí add ‚Üí commit ‚Üí push.

Google Sheets / Docs
Collaboration & Ad-Hoc Analysis
The go-to for quick analysis, one-off reports, stakeholder presentations, and anything that needs real-time collaboration without code.
Use when
  • Quick ad-hoc analysis that doesn't need to be repeatable
  • Sharing data with Sales/Marketing/Finance stakeholders
  • Building models or forecasts (headcount plans, quota math)
  • Meeting notes, project specs, documentation for non-technical teams
üí° If a Sheets analysis keeps getting re-run, that's your signal to move it into Snowflake + Omni instead.
